# Codex 额度耗尽暂停策略

> **本文件不是替代执行方案。**
> 当 Codex 额度耗尽时，QGIS 作图暂停。
> 不启用用户手动操作替代，不启用 PyQGIS 手动粘贴，不启用 qgis_process，不启用 GUI 手动点击。
> 只生成"额度恢复后交给 Codex 执行的任务包"。

---

## Codex 额度状态机

| 状态 | 处理方式 | 是否作图 | 用户角色 |
|------|---------|---------|---------|
| **额度正常** | Claude 生成指令 → Codex computer use 执行 QGIS 操作 | 是 | 提供需求、数据和验收 |
| **额度耗尽** | **暂停作图**，只整理待执行任务包 | 否 | 等待额度恢复 |
| **额度恢复** | 继续交给 Codex 执行待执行任务包 | 是 | 确认结果 |

---

## 额度耗尽时允许做的事（准备性工作）

1. 整理作图需求
2. 检查输入数据清单（文件是否存在、格式是否正确）
3. 规划地图设计参数（标题、图例、比例尺、指北针、标注、配色、CRS、输出格式）
4. 生成 Codex 恢复后可直接执行的任务包
5. 生成待执行清单
6. 记录当前暂停原因和时间

## 额度耗尽时禁止做的事

- 禁止实际作图
- 禁止让用户代替 Codex 操作 QGIS
- 禁止让用户手动执行 PyQGIS 脚本
- 禁止让用户手动点击 QGIS GUI
- 禁止用普通 Python 或 qgis_process 绕开 Codex
- 禁止把未执行的图说成已完成
- 禁止生成伪成果图
- 禁止生成替代执行方案

---

## 待执行任务包

使用 `codex_waiting_task_package_template.md` 模板整理任务信息。

额度恢复后，Codex 根据任务包直接执行，Claude 不再重新生成指令。
