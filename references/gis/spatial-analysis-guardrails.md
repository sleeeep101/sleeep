## 1. CRS 决策框架（不可跳过）

**错误 CRS = 距离/面积/密度计算全部作废。** 做任何空间操作前先检查 CRS。

| 分析涉及… | CRS 要求 | 推荐 |
|----------|---------|------|
| 距离或密度计算 | 投影坐标系（米） | 本地 UTM 或国家网格 |
| 面积计算 | 等面积投影 | Albers / Mollweide / 国家等面积 |
| 仅显示/Web制图 | 任意（地理坐标可） | WGS84 (EPSG:4326) |
| 空间连接/叠加 | 两个图层必须一致 | 先重投影到分析 CRS |
| 混合（距离+显示） | 分析用投影，显示用地理 | 工作流中两个 CRS |

**硬护栏**：如果数据是 EPSG:4326 且分析涉及距离/面积/密度 → **先投影再算**。禁止在经纬度上计算欧氏距离。

**UTM 区域估算**（当没有明显的本地 CRS 时）：
```python
centroid = gdf.geometry.unary_union.centroid
utm_zone = int((centroid.x + 180) / 6) + 1
hemisphere = 'north' if centroid.y >= 0 else 'south'
epsg = 32600 + utm_zone if hemisphere == 'north' else 32700 + utm_zone
```

## 2. 研究问题分类

选方法之前先对研究问题分类：

| 目标 | 信号词 | 典型方法族 |
|------|--------|----------|
| **描述** | "空间格局是什么"、"X 如何分布" | Choropleth / KDE / 汇总统计 / ESDA |
| **解释** | "什么因素解释"、"为什么 X 空间变化" | 回归阶梯 (OLS → 空间 → 局部) |
| **比较** | "X 在区域间是否有差异" | 分层分析 / 交互项 / 区域子集 |
| **预测** | "能否预测"、"X 在哪里发生" | ML + 空间 CV / 特征工程 |
| **聚类/热点** | "事件是否聚集"、"热点在哪里" | Moran's I / LISA / Getis-Ord Gi* / DBSCAN |
| **关联** | "X 与 Y 空间相关吗" | 双变量 Moran's I / 空间回归 |
| **可达性** | "谁有获取…的途径"、"哪些区域服务不足" | 网络分析 / 2SFCA / 等时线 |
| **时空变化** | "X 如何随时间在空间变化" | 面板方法 / 时空聚合 |
| **因果推断** | "X 是否导致 Y" | 空间 DiD / IV / RDD — 明确标注局限 |
| **插值** | "未采样位置的值是多少" | Kriging / IDW / 交叉验证 |

**如果问题映射到多个目标**：确定主要目标，其他作为辅助。问题太模糊 → 要求用户明确结果变量、空间单元和研究区。

## 3. 是否需要空间方法？

不是所有地理数据都需要空间统计：

| 问题 | 如果"否" | 如果"是" |
|------|---------|---------|
| 空间依赖在理论上是否合理？ | 标准方法可能足够 | 空间结构可能重要 |
| 空间结构本身是否是研究问题？ | 空间方法可选 | 空间方法强制 |
| 忽略空间结构是否会偏倚结果？ | OLS 可能足够 | 需要空间调整 |

**准则**：三个都是"否" → 推荐非空间分析并解释原因。不强制在不增值的地方使用空间方法。

## 4. 空间权重选择

| 数据类型 | 权重方案 | 原因 |
|---------|---------|------|
| 规则多边形（网格/栅格） | Rook 邻接 | 边共享 = 邻居 |
| 不规则面（行政区/流域） | Queen 邻接 | 角点接触也算邻居 |
| 点数据 | KNN k=5-8 | 点数不等时 KNN 保证每个单元有相同邻居数 |
| 距离衰减过程 | 距离带 | 指定半径内所有点加权 |
| 网络约束（街道/河流） | 网络距离 | 不能用欧氏距离替代 |

**始终行标准化并检查岛屿**（无邻居的单元）——岛屿会破坏空间统计。

---

## 5. 回归决策阶梯

```
OLS 基线 → 残差 Moran's I 检验 →
  p ≥ 0.05 → OLS 足够，停止
  p < 0.05 → LM 检验 (Lag/Error/Robust) → 
    仅在有理论依据时才进入 GWR/MGWR
```

**硬护栏**：
- GWR 最大 N=5000，MGWR 最大 N=3000（超出→子采样）
- 多重比较用 Bonferroni 或 FDR 校正
- 空间 CV 默认 5 折
- 最大特征数 ≤ 15（超出→降维）
- 默认显著性水平 α=0.05

## 6. 数据质量检查

| 问题 | 何时检查 | 处理 |
|------|---------|------|
| 缺少 CRS | 始终 | 询问用户或从坐标范围推断 |
| 无效几何 | 空间操作前 | `gdf.geometry.buffer(0)` 修复 |
| 重复几何 | 空间统计前 | 先调查上下文再决定是否删除 |
| 多部件要素 | 分析需要单部件时 | `gdf.explode()` |
| 空几何 | 始终 | 删除或调查 |
| 缺失值 | 始终 | 记录缺失的空间格局——聚集的缺失会偏倚空间统计 |

**多数据集集成**：CRS 不匹配→重投影到共同分析 CRS；时间不匹配→假设空间格局稳定，间隔>2年需标记；分辨率不匹配(MAUP)→记录聚合/解聚方法，警告生态学谬误。

## 7. 条件诊断（非固定清单）

诊断（残差 Moran's I、MAUP 敏感性、替代空间权重、GWR/MGWR、空间 CV）**仅在声明依赖它们时才应用**——不是每项分析都要跑全套。添加或跳过重度诊断之前需要人工确认点。

## 8. DEM/地形分析护栏（2026-07-14 补充，night_owl 无此内容，自建）

> night_owl 未覆盖 DEM/地形专用规则。以下基于 GIS 领域标准实践补充。

### DEM 分辨率与分析尺度

| 分析类型 | 最低 DEM 分辨率 | 推荐数据源 |
|---------|:------------:|---------|
| 全球/大陆尺度地形 | 90m-1km | SRTM 90m / ALOS AW3D30 |
| 区域流域分析 | 30m | ASTER GDEM v3 / SRTM 30m |
| 局部地形因子（坡度/坡向/曲率） | 12.5-30m | ALOS PALSAR 12.5m / 无人机 DSM |
| 沟蚀/细沟识别 | ≤5m | 无人机 LiDAR / 立体像对 |
| 水文建模（河道提取） | 30m（最小） | 填洼预处理后的 DEM |

### 填洼（Sink Filling）规则

- 水文分析前**必须**填洼：`r.fill.dir`（GRASS GIS）/ `Fill`（ArcGIS）/ `wang & liu`（Python pysheds）
- 填洼后检查：填洼前后高程差 > 研究区高程范围 5% → 数据质量问题，需人工检查
- 平原区（坡度<2°）：填洼可能导致大面积平坦区，考虑使用高分辨率 DEM 或辅助河道数据

### 地形因子计算要求

- 坡度/坡向：用 Horn 算法（3×3 窗口），不要用简单差分
- 曲率：报告剖面曲率和切面曲率，不要只算总曲率
- TPI（地形位置指数）：必须报告分析窗口半径及选择依据
- 地形湿度指数（TWI）：需要先做填洼 + 流向计算，报告汇水面积单位

### 地形分类

- 地貌分类（geomorphon）：必须报告查找半径（L）和坡度阈值（t）
- 如果结果对参数敏感，必须做参数敏感性分析

## 9. 地图可视化规则（自 night_owl 融合）

### 每张地图必须包含
1. 标题 2. 图例（含单位和分级方案）3. 比例尺 4. CRS 标注 5. 相关地图之间配色一致 6. 指北针和数据源（出版要求时）

### 配色规则

| 变量类型 | 使用 | 禁止 |
|---------|------|------|
| 连续（计数/比率） | `viridis`, `YlOrRd`, `Blues` | rainbow/jet |
| 发散（残差/变化） | `RdBu_r`, `coolwarm`, `PiYG` | 连续色图 |
| 分类（聚类） | `Set2`, `tab10` | 连续色图 |

### 分级方案

| 数据分布 | 最佳方案 |
|---------|---------|
| 大致均匀 | `equal_interval` |
| 偏态（常见） | `quantiles` |
| 多峰/自然分组 | `natural_breaks`/`fisher_jenks` |
