## 一、声明-证据映射（Claim-Evidence Mapping）

**来源**：Nature-Paper-Skills §manuscript-optimizer + paper-writing-skill Principle 1

### 方法论

在写作或修订任何章节前，先建声明-证据映射表：

```markdown
| 声明 | 支撑证据（图/表/补充材料） | 状态 |
|------|--------------------------|------|
| {一句话声明} | Fig X, Table Y, Supp Z | supported / partial / missing |
```

### 操作规则

1. 从摘要、引言、讨论中提取**每条实质性声明**
2. 为每条声明指认精确的支撑证据（图号、表号、补充项）
3. 标记状态：
   - **supported**：证据直接充分支撑
   - **partial**：部分支撑，存在缺口
   - **missing**：无证据支撑
4. 对 `partial` 或 `missing` 的声明，**只允许三种操作**：
   - 弱化声明（降级措辞）
   - 补充缺失证据
   - 将声明重新定性为动机/假设/推测

### 红牌规则

**不允许用更强的修辞掩盖证据缺口。** 如果摘要或引言比 Results 更强，必须修复结构而非修饰文字。

## 二、五层审计优先级

**来源**：Nature-Paper-Skills §manuscript-optimizer

### 优先级排序（自上而下，不可逆序）

```
1. 方向（Direction）     → 核心贡献是否明确？问题是否立得住？
2. 逻辑（Logic）         → 证据链是否完整？声明是否有支撑？
3. 视觉（Visual）        → 图表是否独立可读？图注与正文是否一致？
4. 术语（Terminology）   → 核心概念命名是否稳定？是否有同义词漂移？
5. 语言（Language）      → 语法、句式、格式、标点
```

### 核心纪律

**上层不稳定时，禁止下沉到下层工作。** 不要在一句话的措辞上花时间，如果它承载的声明本身证据不足。

## 三、Introduction-Twice 原则

**来源**：paper-writing-skill Principle 1

### 两次写引言

| 轮次 | 时机 | 目的 | 命运 |
|------|------|------|------|
| **Draft 0** | 实验设计前 | 设定评估护栏，明确论文试图证明什么 | 明确可丢弃，只是脚手架 |
| **Final** | 实验完成后 | 受证据约束，每条声明精确映射到评估结果 | 最终版本 |

### 为什么有效

- 没有 Draft 0：评估没有统一论点导向，实验各自为战
- 不重写最终版：引言承诺了实验无法兑现的声明 → 被拒

### 操作

写论文前先写一段 Draft 0 引言（问题→缺口→粗略贡献），做完实验后**从零重写**引言，不以 Draft 0 为编辑起点。

## 四、What → Why → So-What 标题演进

**来源**：paper-writing-skill Principle 3

### 三阶段

| 阶段 | 特征 | 示例 |
|------|------|------|
| **WHAT**（学生稿） | 描述性，只命名主题 | "实验结果" |
| **WHY**（首次编辑） | 问题驱动 | "为什么线性方法在此失效？" |
| **SO-WHAT**（终稿） | 声明优先，含结论 | "系统 X 在所有基线中以 2-4× 优势胜出" |

### 检验标准

**扫读标题能否重建论文论点？** 如果标题只含主题不含结论 → 扫读者得到零信息 → 需要升级到 SO-WHAT 级。

## 五、图表声明规划

**来源**：Nature-Paper-Skills §figure-planner

### 一图一声明原则

每张主图承载**一个主导声明**。如果一张图不能用一句话概括，要么拆分，要么降级部分面板到补充材料。

### 面板角色分类

在写图注或 Results 段落前，给每个面板分配一个角色：

- **声明支撑证据**（claim-supporting evidence）
- **方法论桥梁/定义**（methodological bridge or definition）
- **新条件下的验证**（validation under a new regime）
- **排序/基准比较**（ranking or benchmark comparison）
- **转化/实践意义**（translational or practical consequence）
- **案例说明**（case illustration）
- **失败模式/局限**（failure mode or limitation）

禁止一个面板同时承担三个角色。

### 主图 vs 补充材料决策

**留在主图**：支撑核心声明必需的面板、读者必须立即看到的关键比较
**移到补充材料**：鲁棒性变体、更密集的方法对比、扩展案例、次要消融

### 图注纪律

图注不止解码坐标轴。每条图注应：
- 定义每个面板的角色
- 保留正文压缩后省略的量化锚点
- 与面板字母、指标、数据集、基线保持完全一致
- 避免比图中数据更强的解释性语言

## 六、七种压缩操作

**来源**：paper-writing-skill §compression_patterns

| # | 操作 | 做法 | 适用场景 |
|---|------|------|---------|
| 1 | **句子缩短** | 删除从句、冗余修饰、清嗓子短语 | 全文始终 |
| 2 | **段落合并** | 同一观点用不同例子的多个段落 → 一个段落+最佳例子 | 例子堆砌时 |
| 3 | **去通用形容词** | "significant""novel""promising" → 具体数字或删除 | 修饰泛滥时 |
| 4 | **删除教程解释** | 删除目标期刊读者已经知道的基础概念解释 | 背景部分过长时 |
| 5 | **转声明优先句式** | 标题含结论非主题，段落开头声明先行证据后随 | 标题和主题句重构 |
| 6 | **评估压缩为 Takeaways** | 多段详细结果 → 分组+一个 Takeaway 段陈述含义 | Results 过长时 |
| 7 | **文字转图表** | 密集数值比较从散文移到表格，散文只解读模式 | 数字罗列时 |

### 压缩强度判断

| 压缩比 | 信号 |
|--------|------|
| 10-20% | 稿子已接近终稿形态 |
| 30-50% | **正常**。作者终于知道什么重要 |
| 50-65% | 危机压缩。叙事框架变了 |
| >65% | 可能是结构性重写 |

## 七、逆向大纲与主题句先行

**来源**：Nature-Paper-Skills §scientific-writing + paper-writing-skill craft_reference

### 逆向大纲（修订现有段落前）

```
1. 写出一节的论点（section thesis，一句话）
2. 为每个段落写一行：
   - 段落任务
   - 段落承载的关键证据或推理
   - 与前一段的过渡关系
3. 合并/移动/删除无法清晰映射到节论点的段落
```

### 主题句先行（起草新段落前）

写完大纲后，先写**所有段落的主题句**，验证它们连贯形成论点链，再展开为完整段落。主题句必须：
- 以声明开头（不是背景、不是上下文）
- 可被扫读理解，独立构成论证

## 八、英文科学散文风格规则

**来源**：Nature-Paper-Skills §scientific-prose-style + paper-writing-skill craft_reference

### 硬规则

| # | 规则 | 操作 |
|---|------|------|
| 1 | **破折号预算** | 每段 ≤1 个 em-dash，摘要 0 个 |
| 2 | **冒号不传染** | 别把破折号全换成冒号——如果每句都以 `: ` 结尾，只是换了种 tic |
| 3 | **修饰一次** | `may possibly suggest that ... could in principle ...` → `These results suggest X` |
| 4 | **句式长短交替** | 一个长句后面跟一个短句 |
| 5 | **段落开头不重复** | 连续段落不以同一个词开头 |
| 6 | **数字替代形容词** | "significant improvement" → "2-4× improvement" |
| 7 | **命名优于泛指** | 每个概念有专名，不出现"data-driven approach"这种万能用语 |
| 8 | **声明优先主题句** | 段落开头直接陈述断言，不铺垫背景 |
| 9 | **主动语态** | `we compiled` 非 `compilation was performed` |
| 10 | **句长均值 ~21 词** | 最长句 ~40 词，留给贡献列表 |

### 自检（提交任何英文段落前）

1. 数破折号：每段超过 1 个或摘要中出现 → 重写
2. 数段落开头词：重复 → 变换
3. 朗读最长句：一口气读不完 → 拆分

## 九、Results 段落架构速查（8 项）

**来源**：Nature-Paper-Skills §results-section-revision

修改 Results 任一小节前，逐项检查：

1. 标题是否在卖结论而非方法？（"X 在 Y 条件下优于 Z" 而非 "X 的性能评估"）
2. 每段首句是否陈述该段论点？
3. `we asked`/`we examined`/`to test this` 是否过度使用？
4. 抽象词是否遮挡了更具体的名词？
5. 与前一段的关系是否明确？
6. 该段是在仅报告结果，还是在推进论证？
7. 段尾是否陈述了解读上的变化？
8. 小节结尾是否自然引出下一小节？

### 小节模式

每个 Results 小节应依次回答两个读者问题：
1. 为什么现在需要这个分析？
2. 它确立了什​​么？

如果第二个问题出现在第一个之前，小节读起来会很突兀。

## 十、论文项目状态文件

**来源**：Nature-Paper-Skills §paper-bootstrap

### 四文件制

| 文件 | 内容 | 更新时机 |
|------|------|---------|
| `project_truth.md` | 论文故事、活跃手稿、核心声明、稳定约束 | 实验或数据变更后 |
| `decision_log.md` | 持久的框架/范围决策，后续修订必须尊重 | 任何重大方向决定后 |
| `result_summary.md` | 实验到手稿的翻译：已验证发现、弱发现、图表锚点 | 每轮实验后 |
| `paper_handoff.md` | 当前写作就绪包：哪些可以起草、哪些缺证据、哪些图表承载哪些论点 | 每次切换写作会话前 |

### 核心原则

**不要凭借记忆写作。** 实验更新后、统计分析后、图表修改后，先刷新这四个文件，再做大量手稿修订。

## 十一、LaTeX 编译诊断与格式审计

**来源**：academic-writing-skills (bahayonghang)

### 编译工作流

1. 编译前运行 `latexmk -pdf -interaction=nonstopmode`
2. 只解释第一个阻塞错误（不跳入后续级联错误）
3. 对每个错误给出：位置（行号）→ 根因 → 修复 → 重新编译
4. 最多 3 轮；超过则请求手动干预

### 中文学位论文专项（GB/T 7714-2015）

- 参考文献格式校验：作者-题名-期刊-年份-卷期-页码 六字段完整性
- 模板兼容性：thuthesis / pkuthss / 等模板的 class 选项冲突检查
- 中英文混排：中文字体 + 英文/数字的西文空格（⅛ em）一致性

### 提交前格式审计项

来源：academic-writing-skills §paper-audit

- 摘要与正文声明一致
- 每个图号在正文中有对应引用
- 每个引用在 `.bib` 中有完整条目
- 无孤立标签（`\label` 无对应 `\ref`）
- 交叉引用完整性（公式号、表号、节号全部对应）

## 精进优先级（建议实施顺序）

| 优先级 | 方法论 | 理由 |
|--------|--------|------|
| **P0 立即可用** | 声明-证据映射、五层审计优先级 | 不依赖任何工具，直接改变写作前的思考结构 |
| **P0 立即可用** | Introduction-Twice | 改变写作顺序即可，保护性极强 |
| **P1 下次写作时用** | 逆向大纲 + 主题句先行、图表声明规划 | 需要在实际写作任务中应用 |
| **P1 下次写作时用** | 英文散文风格 10 规则 + 自检 | 可配合 -ai skill 在英文润色时执行 |
| **P2 写作后** | 七种压缩操作、Results 8 项速查 | 修订阶段工具 |
| **P2 项目级** | 四文件状态系统 | 需要建立项目习惯 |
| **P2 收尾** | LaTeX 编译诊断、GB/T 7714 审计 | 中文核心投稿收尾阶段 |
