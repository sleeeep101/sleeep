# 术语账本方法

> 来源：nature-skills `_shared/core/terminology-ledger.md`
> 用途：论文写作质量核验中「术语统一」维度的操作方法

## 核心原则

一篇论文中，同一个方法/模型/数据集/指标/概念，在全文必须使用同一个名称。
术语在不同章节之间漂移，审稿人会解读为粗心。
在开始起草或润色之前建立术语账本，作为全文的唯一真相源。

## 操作流程

### 1. 初稿时建立术语账本

在收到手稿/草稿/笔记后，先提取所有重复出现的领域术语：

- 方法、模型、系统、算法、模块、框架
- 数据集、基准、队列、材料
- 指标、单位、统计符号、数学符号
- 缩写词和首字母缩略词（含完整形式）
- 论文定义或反复依赖的关键概念

对每个术语记录：规范形式、首次使用的展开形式（缩写）、源文件中已有的变体。

### 2. 向用户呈现术语账本

在第一个输出之前或同时，展示一个紧凑表格：

| 规范术语 | 首次定义 | 源文件中存在的变体 | 决策 |
|---------|---------|-------------------|------|
| 数字高程模型（DEM） | Digital Elevation Model（DEM） | "DEM数据"、"高程模型"、"数字地形模型" | 首次展开后统一使用"DEM" |
| RUSLE | Revised Universal Soil Loss Equation（RUSLE） | "RUSLE模型"、"修正通用土壤流失方程" | 首次展开后统一使用"RUSLE" |

标记每次冲突：同一概念用了不同名称，或同一名称被重用于两个不同概念。

### 3. 锁定并执行

一旦术语固化，后续所有编辑和润色都必须遵守。不允许引入新的变体。如果必须更改规范形式，更新术语账本并全文替换。

## GIS/遥感/DEM 领域常见术语冲突

| 易混淆术语 | 区分 |
|-----------|------|
| DEM / DTM / DSM | DEM = 数字高程模型（裸地面），DSM = 数字地表模型（含植被建筑），DTM = 数字地形模型（含地形特征） |
| 地形因子 / 地形属性 / 地形参数 | 统一用"地形因子"，首次定义时列出包含的具体因子 |
| 土壤侵蚀 / 水土流失 / 土壤流失 | 中文语境下需明确区分：土壤侵蚀 = 广义概念，水土流失 = 水蚀过程 |
| 空间分析 / 空间建模 / 空间统计 | 空间分析 = 总称，空间建模 = 构建模型，空间统计 = 统计方法子集 |
| 遥感影像 / 遥感图像 / 卫星影像 | 统一使用"遥感影像" |
