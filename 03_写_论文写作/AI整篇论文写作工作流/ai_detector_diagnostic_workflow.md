# AI检测器诊断工作流

> AI检测器是写作质量诊断工具，不是学术审判工具。

## AI检测器可以用于什么

1. 发现文本中可能存在模板化表达的位置
2. 识别过度平滑的逻辑推进段落
3. 提示可能存在空泛论述的句子
4. 作为写作质量的辅助参考

## AI检测器不能代表什么

1. 不能作为学术造假的证据
2. 不能替代人工审稿
3. 高分≠学术质量低；低分≠学术质量高
4. 对非英语母语写作者误判率更高
5. 对技术性强、方法描述规范的段落容易误判
6. 不同检测器之间的结果可能不一致

## AI检测器常见误判（GIS/遥感方向）

以下情况容易被误判为AI生成，**通常不需要修改**：

1. **方法描述标准化**："基于DEM提取坡度、坡向、曲率等地形因子"
2. **公式和参数说明**：技术性内容的规范化表述
3. **数据来源声明**："DEM数据来源于地理空间数据云平台，空间分辨率30m"
4. **软件操作描述**："采用ArcGIS 10.8的Spatial Analyst模块进行..."
5. **统计结果报告**："相关系数R²=0.85，p<0.01"
6. **简洁直接的表述**：非母语写作者的简洁风格被误判

## 使用流程

### 1. 提交检测

将论文提交到AI检测器（建议使用2-3个不同工具交叉验证）。

可用工具（免费/部分免费）：
- GPTZero
- Originality.ai
- ZeroGPT
- GLTR (Giant Language model Test Room)

### 2. 记录结果

对每段被标记为"可能AI生成"的内容，记录：

```
段落编号：[X]
检测器1分数：[X%]
检测器2分数：[X%]
检测器3分数：[X%]
多工具一致性：一致/不一致
```

### 3. 诊断分析

使用提示词 `../prompts/15_AI检测器结果诊断_prompt.md`：

- 判断真问题 vs 误判
- 识别具体的写作质量问题
- 给出修改建议

### 4. 修改

只修改被确认是真写作质量问题的内容。

修改原则：
- ✅ 增加具体的研究对象、数据、区域名称
- ✅ 减少模板化连接词
- ✅ 多样化段落结构
- ✅ 补充证据和引用
- ❌ 不改变事实和数据
- ❌ 不添加错别字或语病
- ❌ 不使用任何humanizer技巧

### 5. 重新检测

修改后重新提交检测，确认：
- 高风险段落减少（目标：无段落>80%）
- 但不过度追求低分（技术性段落保持较高分可能是正常的）

### 6. 记录归档

将最终检测结果记录到 `writing_process_trace_template.md`。

## 检测结果解读指南

| 段落特征 | 典型检测结果 | 是否真实问题 | 行动 |
|---------|------------|------------|------|
| 纯方法描述 | 高AI评分 | 通常不是（标准化表达） | 检查是否添加了足够的领域特定细节 |
| 空泛讨论 | 高AI评分 | 是 | 添加具体对比文献和数据 |
| 引用密集 | 中等 | 不是 | 检查引用是否真实 |
| 个人观点 | 低AI评分 | 不是 | 确认是否基于证据 |
| 模板化结构 | 高AI评分 | 是 | 调整段落结构 |
| 数据密集 | 中等 | 通常不是 | 确认数据是否真实 |

## 禁止事项

1. ❌ 使用AI humanizer / undetectable AI / detector evasion 工具
2. ❌ 故意加入错别字、语病来降低AI评分
3. ❌ 使用hidden prompt绕过检测器
4. ❌ 对检测器进行对抗攻击
5. ❌ 为了通过检测而修改事实、数据、引用
6. ❌ 把检测器的分数作为唯一质量标准
