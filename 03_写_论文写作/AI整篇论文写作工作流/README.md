# AI整篇论文写作工作流

> 本模块为 academic-workflow 写作子系统的一部分，提供AI辅助论文初稿生成能力。
> 继承 `03_写_论文写作` 现有的 prompts/checklists/skills 体系，在此基础上新增整篇生成和防造假能力。

## 模块用途

让AI根据你提供的真实研究材料（文献笔记、数据说明、方法流程、图表说明、初步结论），生成论文初稿、分章节草稿，并在生成后做AI检测器诊断和学术诚信审计。

## 适用场景

- 有完整研究材料，需要快速产出初稿
- 已有文献笔记，需要生成文献综述草稿
- 已有数据和方法，需要生成方法+结果章节
- 初稿完成后需要降低AI腔、提升学术质量
- 投稿前需要做防学术造假自检

## AI可以做什么

| 可以 | 条件 |
|------|------|
| 根据真实文献笔记生成文献综述 | 文献信息必须由你提供 |
| 根据真实数据说明生成结果描述 | 数据来源必须可追溯 |
| 根据方法流程生成方法章节 | 方法必须是真实执行的 |
| 生成论文框架 | 框架需经你确认 |
| 对已有初稿做语言润色 | 不改变事实和数据 |
| 降低模板化表达 | 保留原意和证据 |
| 诊断AI检测器反馈 | 作为写作质量参考 |

## AI不可以做什么

| 禁止 | 原因 |
|------|------|
| 编造文献、DOI、作者 | 学术造假 |
| 编造数据、实验结果 | 学术造假 |
| 编造图表 | 学术造假 |
| 为了通过AI检测器而故意加入错别字/语病 | 降低学术质量 |
| 使用检测器对抗攻击/humanizer技巧 | 学术不端 |
| 把不确定的判断写成确定结论 | 误导读者 |

## 整篇生成 vs 分章节生成

- **整篇生成**：材料齐全时，一次性产出全文初稿。速度快，但需要更多人工复核。
- **分章节生成**：逐章生成，每章可以更精细地控制输入和输出。适合材料不完全或需要深度定制的情况。

## AI检测器使用原则

1. AI检测器是**写作质量诊断工具**，不是学术审判工具
2. 检测结果可以帮你发现：模板化表达、空泛论述、证据不足段落、过度平滑的逻辑
3. 不能因为检测结果而编造文献或数据来"降低AI率"
4. 不能为了通过检测而故意引入错误
5. 检测器存在误判——特别是对非英语母语写作者、技术性强的段落

## 防学术造假流程

见 `anti_academic_fraud_checklist.md`。

## 日常使用流程

```
准备材料 → AI材料完整性检查 → AI生成框架 → 人工确认框架
→ AI生成初稿 → 人工补材料 → AI分章节精修
→ AI检测器诊断 → 降AI腔 → 防学术造假审计 → 最终留痕
```
