# 整篇论文生成工作流

> 9步工作流：从材料准备到最终留痕

## 前置条件

开始之前，确认以下事项：
- [ ] 你的研究问题已经明确
- [ ] 你有真实的数据（或知道数据在哪里获取）
- [ ] 你大致知道用什么方法
- [ ] 你有至少5-10篇核心文献的笔记
- [ ] 你理解：AI生成的是**初稿**，最终论文需要你逐句验证和修改

## 9步工作流

### 步骤1：材料准备

准备以下材料清单（打勾表示已有）：

```
□ 研究题目（或有2-3个候选）
□ 研究问题/研究目标（1-2句话）
□ 研究区基本信息（位置、地形、气候、土地利用等）
□ 数据来源清单（类型、分辨率、来源、获取时间）
□ 方法流程说明（技术路线图 + 关键步骤描述）
□ 图表说明（编号、标题、每个图/表展示什么）
□ 文献笔记（每篇含：作者/年份/核心观点/DOI/与你研究的关系）
□ 初步结论/研究发现（逐条列出，含关键数字）
```

### 步骤2：AI材料完整性检查

使用提示词 `13_整篇论文初稿生成_prompt.md` 的第一步，让AI检查材料完整性。

输出：[材料完整性诊断]

### 步骤3：AI生成论文框架

AI基于现有材料生成章节结构。你逐项确认或调整。

输出：[论文框架]

注意：框架阶段就要标记哪些节材料不足。

### 步骤4：AI生成整篇初稿

使用提示词 `13_整篇论文初稿生成_prompt.md` 的完整流程。

AI输出：
- 完整论文初稿
- 【需补充】标记
- 【需人工确认】清单

### 步骤5：人工补材料

根据AI输出的【需补充】清单，补充：

1. 缺失的文献（用Zotero/CNKI查找）
2. 缺失的数据（如果数据不可得，调整研究范围）
3. 缺失的图表（从真实数据生成）
4. 缺失的方法细节（回顾实验/分析记录）
5. 缺失的结论依据（重看数据和结果）

### 步骤6：AI分章节精修

补充材料后，对每个章节单独精修：

| 章节 | 使用提示词 | 需要提供的额外材料 |
|------|-----------|------------------|
| 引言 | 05_literature_review_builder.md | 文献笔记 |
| 文献综述 | 05_literature_review_builder.md | 文献笔记 + 综述框架 |
| 研究区 | 06_abstract_intro_method_result_discussion.md | 研究区数据 |
| 数据与方法 | 06_abstract_intro_method_result_discussion.md | 数据清单 + 方法流程 |
| 结果 | 06_abstract_intro_method_result_discussion.md | 所有图表 + 关键数字 |
| 讨论 | 06_abstract_intro_method_result_discussion.md | 对比文献 + 局限性说明 |
| 结论 | 06_abstract_intro_method_result_discussion.md | 主要发现逐条列表 |

### 步骤7：AI检测器诊断

1. 将精修后的论文提交到AI检测器（GPTZero/Originality等）
2. 记录每个段落的检测结果
3. 使用提示词 `15_AI检测器结果诊断_prompt.md` 分析结果
4. 区分真问题 vs 误判
5. 对确认为真问题的段落进行修改

注意：只改写作质量问题，不改事实、数据、引用。

### 步骤8：防学术造假审计

使用提示词 `16_防学术造假审计_prompt.md` + `anti_academic_fraud_checklist.md`：

1. 文献真实性检查
2. 数据来源检查
3. 图表对应检查
4. 方法可执行性检查
5. 结果—方法对应检查
6. 结论边界检查
7. AI生成内容标记

任何🔴红旗 → 立即修正后再进入步骤9。

### 步骤9：最终版本留痕

填写 `writing_process_trace_template.md`：

- 记录AI参与的范围和程度
- 记录检测器诊断结果
- 记录所有人工修改
- 生成最终版本说明

---

## 重要提醒

1. **AI生成的是初稿，不是终稿**：每次都需人工审核每个事实、引文和数据
2. **材料不全不要硬生成**：缺失材料的地方标记【需补充】，不要编造
3. **写不出来可能是研究没做够**：如果AI也无法生成合理的内容，可能说明你的研究本身有gap
4. **导师意见优先**：AI工作流的任何产出最终都要经过导师确认
