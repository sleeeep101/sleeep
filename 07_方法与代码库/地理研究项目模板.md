# 地理研究项目模板

适用于 GIS、遥感、DEM、空间统计和地理机器学习项目。创建新项目时可按下列边界组织；目录可按项目规模裁剪，但不要混合原始数据、代码、结果和个人配置。

```text
project/
├── README.md                 # 问题、研究范围、运行方式、限制
├── LICENSE                   # 发布时自行选择
├── .gitignore                # 数据、输出、日志、密钥和本机配置
├── requirements.txt          # 或 pyproject.toml / environment.yml
├── configs/                  # 可公开的参数模板，不放本机绝对路径
├── data/
│   ├── external/             # 第三方数据的来源说明或下载脚本
│   ├── raw/                  # 不改写；通常不提交
│   └── processed/            # 可重建；通常不提交
├── scripts/                  # 下载、预处理、分析、制图等可执行入口
├── src/                      # 可复用方法实现
├── tests/                    # 最小单元或冒烟测试
├── references/               # 数据字典、方法说明、许可说明
└── outputs/                  # 图、表、日志和报告；通常不提交
```

## 方法证据卡

每个进入 `src/` 或 `scripts/` 的方法都应记录：

| 字段 | 必填内容 |
|---|---|
| 问题与尺度 | 研究对象、空间/时间尺度和适用边界 |
| 输入 | 数据来源、分辨率、坐标系、许可与缺失值处理 |
| 方法 | 可复现步骤、参数和随机种子 |
| 输出 | 图层、表格、指标及单位 |
| 验证 | 基准、交叉验证、敏感性或人工核验方式 |
| 局限 | 不适用情形、潜在偏差和未验证假设 |
| 复现命令 | 从干净环境运行的最短命令 |

## 发布前硬性检查

- [ ] 没有绝对用户路径、密钥、Cookie、账号、真实联系人或未公开研究数据。
- [ ] 原始数据可由公开来源重新获取，或明确其不可分发原因。
- [ ] 主图和主表能追溯到脚本、配置、输入版本和运行环境。
- [ ] `README.md` 说明安装、运行、预期输出、失败模式和许可证。
- [ ] 至少有一个不依赖私人数据的测试或合成示例。
- [ ] 结论与代码产出相符；没有把未验证推断写成事实。
