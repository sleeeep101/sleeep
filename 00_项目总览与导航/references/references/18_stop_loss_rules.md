# 18_stop_loss_rules — 停损机制

防止 agent 工作流越做越大、越跑越乱。每个投入方向都要有止损线。

---

## 1. 论文阅读停损

| 触发条件 | 止损动作 |
|---------|---------|
| 一篇论文 30 分钟内看不出价值 | 只保留题录，不继续精读 |
| 论文没有全文，只能基于摘要 | 不能写成确定结论，标注「仅基于摘要」 |
| 方法太复杂且当前阶段用不上 | 标注「后期再读」，不入长期知识库 |
| 与 GIS/遥感/空间分析完全无关 | 除非方法迁移价值 ≥ 8，否则不精读 |
| 连续 3 篇论文读不懂 | 暂停该方向，切换更容易的方向 |
| 读完 3 篇同一子方向论文后感觉没有新信息 | 该子方向暂缓，等有新方法再回来 |

---

## 2. GitHub Skill 调研停损

| 触发条件 | 止损动作 |
|---------|---------|
| 一个 GitHub skill 看 10 分钟没有独特结构 | 不整合，只记录名称和一句话描述 |
| 没有 license 或 license 不清楚 | 不复制原文，只记录来源和结构 |
| 功能与已有 skills 高度重复 | 只记录不加入主流程，标注「已有 X 替代」 |
| 第一轮调研到达 30 个高相关仓库 | 停止扩展，输出调研报告 |
| 调研总时间超过 3 小时 | 停止本轮，等待下一轮计划 |

---

## 3. Codex / AI Code 停损

| 触发条件 | 止损动作 |
|---------|---------|
| 同一个问题连续修 3 次失败 | **暂停**，不再继续让 agent 猜 |
| 3 次失败后 | 写入 `failed_attempts.md` |
| 无法继续推进 | 降级为说明文档或流程笔记 |
| 缺少关键数据 | 标注「等待数据」，暂不开发 |
| 环境配置超过 1 小时无法跑通 | 记录环境问题，换工具或简化方案 |

---

## 4. GIS 项目停损

| 触发条件 | 止损动作 |
|---------|---------|
| 一个 GIS 小工具超过 2 天不能跑通 MVP | 降级为操作流程记录 |
| 全自动作图类任务 | 必须先降级为固定模板半自动流程 |
| 没有数据源 | 不进入正式开发，标注「等待数据」 |
| 依赖付费软件且无替代 | 标注「需 ArcGIS/QGIS 手动操作」，不写成自动化代码 |

---

## 5. 选题停损

| 触发条件 | 止损动作 |
|---------|---------|
| 找不到数据的选题 | 暂缓，标注「数据不可得」 |
| 需要过高数学/编程能力 | 标注「后期再说」 |
| 不能服务毕业或 GIS 主线 | 不优先推进，放入选题池底 |
| 选题超过 3 个月没有进展 | 复盘：是否应该放弃 |

---

## 6. 知识库停损

| 触发条件 | 止损动作 |
|---------|---------|
| 长期知识库超过 200 篇论文卡片 | 启动季度瘦身，归档低价值内容 |
| 方法迁移库同一方法出现 5 次以上 | 合并为一条综合条目 |
| 写作句式库超过 100 条 | 按场景分类整理，删除重复 |
| 选题池超过 20 个候选 | 筛选 TOP 5，其余入暂缓 |

---

## 7. 每日/每周停损

| 触发条件 | 止损动作 |
|---------|---------|
| 每日搜索论文超过 30 分钟 | 停止搜索，用现有结果 |
| 每日精读超过 3 篇 | 不再新增，把当天的读完就好 |
| 每日 agent 任务超过 3 个 | 不再新增，等现有完成 |
| 每周 AI Code 任务积压超过 5 个 | 暂停新增，先清旧任务 |

## 停损判断速查

| 场景 | 信号 | 动作 |
|------|------|------|
| 论文阅读 | 30分钟无价值 | 保留题录，停止 |
| Skill调研 | 10分钟无独特结构 | 记录不整合 |
| Codex | 3次失败 | 暂停+记录+降级 |
| GIS项目 | 2天无MVP | 降级为操作记录 |
| 选题 | 3月无进展 | 复盘放弃 |
| 知识库 | 200+卡片 | 季度瘦身 |
