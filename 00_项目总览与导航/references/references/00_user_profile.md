### B. GIS/遥感就业相关

### C. 方法迁移 (从任何学科迁移到 GIS)
- 机器学习 / 深度学习 / GeoAI
- 时空建模 / 空间统计 / 因果推断
- 风险评估 / 生态环境评价 / 指标体系

### D. 学术写作
- 文字能力偏弱，需要系统沉淀
- 需要：句式模板、段落结构、写作逻辑、中英文学术表达

## 偏好与约束

1. **不强行迁移** — 论文方法优先判断 GIS/遥感价值，不强行迁到 ETF 量化/3D/LifeOS 等个人项目
2. **低基础友好** — 遇到复杂方法要给初学者解释版
3. **Markdown 化** — 所有产出尽量 Markdown，方便 agent 后续读取
4. **防编造** — 不确定就标"未确认"，不编造 DOI/数据/引文/结果
5. **MVP 优先** — 复杂任务先做最小可用版本，不过度扩展
6. **长期可读** — 输出要能沉淀到知识库，几个月后还能看懂
