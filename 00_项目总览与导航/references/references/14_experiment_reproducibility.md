## 数据版本管理

每次数据变化（预处理/过滤/增强）都要记录：
```markdown
## 数据集: {数据集名}
- v1: {日期} — 原始数据，{来源}，{条数/大小}
- v2: {日期} — {改动，如"移除异常值"}, {条数变化}
- v3: {日期} — {改动}
```

## 防数据泄露检查

- [ ] 训练/验证/测试集是否时间独立？(时序任务)
- [ ] 训练/验证/测试集是否空间独立？(空间任务)
- [ ] 是否在划分前做了归一化？（错误：应该先划分再归一化）
- [ ] 是否使用了未来信息预测过去？
- [ ] 验证集是否参与了任何特征选择/超参数调优？

## 代码规范

- 训练脚本和推理脚本分离
- 配置文件独立（.yaml/.json）
- README 写清：环境、数据路径、运行命令、预期结果
- 随机种子全部固定
