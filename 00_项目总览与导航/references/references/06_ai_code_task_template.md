## 预置任务：citation-verifier（引用验证脚本）

来源：PaperOrchestra (Levenshtein 模糊匹配) + OpenDraft (四源交叉验证)。

### 任务包

```
# Codex 接手任务包

## 背景
学术工作流中 Claude 不能验证引用的真实性。需要一个确定性脚本验证论文引用是否真实存在。

## 输入
- 论文卡片中的 DOI/arXiv ID/标题列表（JSON）
- 或 LaTeX 的 .bib 文件

## 目标输出
一条可独立运行的 Python 脚本 `citation_verifier.py`

## 执行步骤
1. 读取输入（JSON 或 .bib）
2. 对每条引用，按优先级查询：
   a. Semantic Scholar API（标题模糊匹配 + 年份对齐）
   b. CrossRef API（DOI 精确匹配）
   c. arXiv API（arXiv ID 精确匹配）
3. 计算 Levenshtein 距离（阈值 > 0.70）
4. 输出验证报告：
   - ✅ 已确认：标题/作者/年份/DOI 完全匹配
   - ⚠️ 部分匹配：标题相似但需人工确认
   - ❌ 未找到：三个源都未找到 → 疑似虚构引用
   - 🔍 孤引用：在 .bib 中但正文未引用

## 验收标准
- [ ] 能读取 .bib 和 JSON 两种格式
- [ ] 支持 Semantic Scholar + CrossRef + arXiv 三源查询
- [ ] Levenshtein > 0.70 判定为候选匹配
- [ ] 输出清晰的三态报告
- [ ] 速率限制：≤ 1 QPS（Semantic Scholar 限制）
- [ ] 短标题旁路：标题 < 20 字符时用子串匹配防假阴性

## 不要改的内容
- 不修改输入的 .bib 或 JSON 文件
- 不自动替换引用

## 预期交付
- citation_verifier.py
- README.md（环境依赖 + 使用说明）
```

### 难度：中 | 预估：2-3h | GIS 价值：6 | 优先级：中

## Codex 执行纪律

> 来源：`论文阅读/templates/02_AI_Code任务模板.md`，2026-05-30 迁移

1. **先读文件** — 执行前读完所有相关文件
2. **先理解项目结构** — 用 CodeGraph 或 rg 理解调用关系
3. **最小改动** — 只改必要的行，不顺手重构
4. **不删除日志** — 保留所有日志输出
5. **不删除重试逻辑** — 保留错误处理和重试
6. **不大改外部接口** — 不改变函数签名和模块边界
7. **不自动调权** — 不修改模型参数/权重/因子系数
8. **不自动交易** — 不执行任何交易/下单/调仓逻辑
9. **不夸大模型效果** — 输出中不出现"稳赚""高收益"等
10. **无法验证必须明说** — 如果无法运行或验证，明确说明原因
