# 24 日报质量检查规则

> 最后更新: 2026-07-13
> 基线: 2026-07-13 日报（1,566行 / 62张卡片 / 100%覆盖 / KB入库62篇）

## 检查时间判断

- 当前时间 < 12:00：日报未到运行时间，不算异常
- 当前时间 >= 12:00：检查日报是否生成
- 当前时间 >= 12:15 仍无日报：异常（脚本可能失败）

## 质量基线（2026-07-13 标准，不可降级）

| 检查项 | 最低标准 | 7.13 基线值 |
|--------|---------|:-----------:|
| 日报文件大小 | ≥ 100 KB | 198 KB |
| 日报行数 | ≥ 800 行 | 1,566 行 |
| 知识卡片数 | = 论文总数（100%覆盖） | 62 张 |
| 每张卡片行数 | ≥ 15 行（11字段完整） | ✓ |
| 长期知识库入库 | ≥60分全部入库 | 62 篇 |
| 禁用表达残留 | = 0 | 0 |
| 桌面PDF清零 | 桌面无残留 PDF | ✓ |
| PDF归档 | fulltext_papers/{date}/ 三级目录数量=处理总数 | ✓ |

## 内容质量分级

| 检查项 | 达标 | 未达标 |
|--------|------|--------|
| 知识卡片覆盖 | 全部论文有完整卡片（≥15行） | 任一篇缺卡或短于15行 |
| 7维评分 | 全部论文已评分 | 任一篇未评分 |
| KB入库 | 评分≥60的论文100%入库 | 任一篇≥60未入库 |
| 禁用表达 | BANNED_EXPRESSIONS 残留=0 | 残留 > 0 |
| 归档完整性 | fulltext_papers 三级目录文件数=处理总数 | 数量不一致 |
| "仅基于摘要" | 不存在（全量PDF_TEXT_FULL） | 存在任何摘要级卡片 |
| Agent 卡片质量 | 所有Agent卡片不短于Claude手动卡片 | 明显短于 |

## "仅基于摘要"特殊规则（已淘汰）

> 7.13 基线要求全部论文达到 PDF_TEXT_FULL，不存在"仅基于摘要"的卡片。
> 若出现编码损坏无法提取的PDF，标注"编码质量差"并尽力提取，不跳过。

## 核验流程（每日必做）

1. 台账数量 = 实际桌面PDF数？
2. 日报知识卡片数 ≥ PDF数？
3. KB入库数 = 日报中评分≥60的论文数？
4. fulltext_papers/{date}/ 三个目录文件数之和 = 处理总数？
5. 桌面PDF残留 = 0？
6. BANNED_EXPRESSIONS 残留 = 0？

## 核验输出格式

```
日报：[日期]
总行数：[N] | 文件大小：[N] KB
知识卡片：[N] 张 / 覆盖 [N] 篇（[X]%）
评分分布：95分=[N] | 80分=[N] | 60分=[N] | <60=[N]
KB入库：[N] 篇（≥60分）
禁用表达残留：[N]（预期0）
桌面PDF残留：[N]（预期0）
归档目录：源数据=[N] | 重复=[N] | 未入库=[N] | 合计=[N]
状态：[达标/未达标]
```
