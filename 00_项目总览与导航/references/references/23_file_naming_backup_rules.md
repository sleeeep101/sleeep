# 23_file_naming_backup_rules — 文件命名与备份规则

防止项目、论文、数据、知识库长期积累后混乱，也避免 agent 修改后找不到旧版本。

---

## 一、文件命名规范

### 推荐格式
```
YYYY-MM-DD_主题_类型_v版本号.md
```

### 示例
```
2026-06-25_DEM地形分析_论文卡片_v1.md
2026-06-25_ArcGIS裁剪操作_操作记录_v1.md
2026-06-25_土壤侵蚀RUSLE_方法卡片_v1.md
2026-06-25_论文雷达_日报_v1.md
2026-06-25_黄土高原沟谷提取_codex任务_v1.md
2026-06-25_写作句式_引言背景_v1.md
```

### 命名要素

| 要素 | 格式 | 示例 |
|------|------|------|
| 日期 | YYYY-MM-DD | 2026-06-25 |
| 主题 | 中文简短描述 | DEM地形分析 |
| 类型 | 论文卡片 / 方法卡片 / 操作记录 / 日报 / codex任务 / 周总结 / 句式 | 论文卡片 |
| 版本 | v1 / v2 / final / archived | v1 |

### 禁止的命名方式
- ❌ `新建文件夹/`、`未命名.md`、`111.md`
- ❌ 只有日期没有主题：`2026-06-25.md`
- ❌ 纯英文和拼音混合：`DEM fenxi v1.md`
- ❌ 文件名含空格或特殊字符（URL 中会变成 %20）

## 三、版本规则

| 版本标识 | 含义 | 使用场景 |
|---------|------|---------|
| v1 | 初始版本 | 第一次创建 |
| v2, v3, ... | 重大修改 | 内容有实质变化 |
| final | 阶段性定稿 | 确认不再改动的版本 |
| archived | 已归档 | 不再使用但保留 |
| draft | 草稿 | 还在写、不完整 |

---

## 四、备份规则

### 备份频率

| 频率 | 备份内容 |
|------|---------|
| 每周 | ResearchOS 文件夹、长期知识库.md、方法迁移库.md、写作句式库.md、AI_Code任务池.md |
| 每月 | 完整 ResearchOS + 论文 PDF + 代码项目 + 重要 prompt |
| 重大修改前 | 被修改文件的单文件备份 |

### 备份方式
- 本地备份：复制到 `backups/YYYY-MM-DD_备份/`
- 云备份：复制到 OneDrive 或移动硬盘
- 不依赖单一备份方式

---

## 五、Agent 修改规则

当 agent（Claude/Codex）修改重要文件时，必须遵守：

1. **修改前先备份** — 在 `backups/` 中保存旧版本
2. **不直接覆盖长期知识库** — 生成新条目 → 人工确认 → 追加
3. **大改前说明修改范围** — 改了哪些文件、为什么改
4. **修改后生成 change_log** — 记录修改内容
5. **不确定时生成新版本** — 不覆盖旧版本（v1 → v2）

---

## 六、change_log 模板

```markdown
## Change Log

- **日期**: YYYY-MM-DD
- **修改文件**: {文件路径}
- **修改原因**: {为什么需要改}
- **修改内容**: {具体改了什么}
- **是否影响长期知识库**: 是 / 否
- **是否需要人工检查**: 是 / 否
- **旧版本备份**: backups/YYYY-MM-DD_备份/
- **下一步**: {接下来要做什么}
```

## 七、知识库文件命名特殊规则

| 文件类型 | 命名规则 |
|---------|---------|
| 长期知识库.md | 固定名称，不版本号，靠 change_log 跟踪 |
| 方法迁移库.md | 固定名称 |
| 写作句式库.md | 固定名称 |
| AI_Code任务池.md | 固定名称 |
| 选题池.md | 固定名称 |
| 术语概念库.md | 固定名称 |
| 论文卡片 | `YYYY-MM-DD_主题_论文卡片_v1.md` |
| 日报 | `YYYY-MM-DD_论文阅读日报_v1.md` |
| 周总结 | `YYYY-WXX_周总结_v1.md` |

固定名称的文件（库文件）不版本号，内部通过 change_log 条目追踪修改历史。
