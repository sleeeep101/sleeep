## academic-workflow 的定位

**academic-workflow = 学术执行层。**

负责：论文检索 → 阅读 → 知识卡片 → 方法积累 → 写作 → 组会 → 图表 → 自动化

**不负责：** 总控决策（→ Personal-Brain）、项目数据/实验/结果（→ 具体项目文件夹）、导师沟通记录（→ Personal-Brain）、作品集索引（→ Personal-Brain/career/）

### 与 Personal-Brain 的边界

| 层 | 目录 | 存什么 |
|----|------|--------|
| **总控层** | `Personal-Brain/` | 规则、策略、控制状态、索引、审计、职业规划、压缩后的结论 |
| **执行层** | `academic-workflow/` | 论文源文件、日报、知识库、写作模板、组会材料、图表工具、自动化脚本 |
| **项目层** | `projects/<项目名>/` | 数据、实验、结果、图件、论文草稿、PPT 终稿 |

**核心原则：**
- Personal-Brain 只接收压缩后的状态、索引和结论，不存论文细节
- academic-workflow 不存导师沟通记录和作品集总索引
- 数据/实验/结果/图件跟着具体项目走，不散落在 academic-workflow 根目录

## 各板块详细说明

### 1. 选题与问题库 — 已有毕业要求基线

**当前内容：**
- `../05_选题与问题库/毕业要求_学位论文与发表论文.md` — 大论文格式流程 + 小论文发表期望 + 期刊分级 + 时间约束（2026-07-19 入库）
- `references/15_gap_analysis.md`（研究空白分析方法）
- `references/20_advisor_topic_fit.md`（导师课题适配规则）

**后续：** 入学后选题达 10 个以上时，新建 `选题与问题库.md` 文件（不建新目录）。

### 2. 论文阅读库 — 核心板块（已有）

**这是 academic-workflow 最核心的板块。** 包含论文检索 → 筛选 → 源文件归档 → 日报 → 长期知识库 → 阅读复盘 → 待补读追踪的完整链条。

**对应路径和入口：**

| 内容 | 路径 | 说明 |
|------|------|------|
| 论文检索+筛选+日报脚本 | `scripts/daily_paper_curator.py` | 1722行，每日12:00自动运行 |
| 论文源文件归档 | `paper_sources/` | ~85篇论文，metadata.json+note.md+PDF |
| 每日日报 | `reports/daily/` | 每日自动生成 |
| 全文 PDF | `data/fulltext_papers/` | 已下载的 PDF/HTML 全文 |
| 长期知识库 | `长期知识库.md` | 6字段压缩格式+阅读证据分级 |
| 待补读队列 | `待补读队列.md` | 待获取全文的论文追踪 |
| 去重数据库 | `logs/paper_tracker.db` | SQLite，3张表 |
| PDF下载+解析 | `scripts/fulltext_utils.py` | 5级阅读证据分级 |
| 源文件工具 | `scripts/paper_source_utils.py` | 论文源文件保存/索引 |
| 历史回填 | `scripts/backfill_paper_sources.py` | 历史论文回填 |
| 知识库审计 | `scripts/audit_knowledge_base_reading_level.py` | 知识库质量审计 |
| 论文筛选启动 | `scripts/run_paper_curator.bat` | Windows 批处理启动 |

**重要：**
- 读论文后的复盘在 `paper_sources/<日期>/<论文>/note.md` 中
- 长期知识积累在 `长期知识库.md` 中
- **不单独拆出"复盘库"或"长期知识积累库"板块**

### 3. 数据资源库 — 暂缓

**为什么暂缓：** 当前无实际数据积累。数据应跟着具体科研项目走（`projects/<项目名>/data/`），不在 academic-workflow 根目录建数据目录。`data/fulltext_papers/` 是论文全文，不是通用数据资源库。

入学后可在 Personal-Brain 中建一个 `数据资源索引.md` 指向各项目数据。

### 4. 方法与代码库 — 部分已有（用索引文件启动）

**当前内容：**
- `长期知识库.md` 中的"方法"字段和"可迁移到 GIS/遥感/空间分析的点"字段
- `scripts/` 中的可复用脚本
- `tools/origin_auto_plot/`
- `方法库索引.md`（本次新建）

**方法库索引.md 定位：** 从长期知识库的"方法"字段中提取可复用方法，按统一模板整理。不是独立的知识库，是长期知识库中方法的**可检索索引**。

**收录标准：** 方法流程清晰、可操作、与 GIS/遥感/空间分析相关、已读全文或全文解析达标。
**暂不收录：** 仅摘要级、与 GIS/遥感无关、方法描述过于模糊。

### 5. 作图与可视化库 — 部分已有

**当前内容：**
- `tools/origin_auto_plot/` — Origin 自动绘图桥接
- `prompts/origin_auto_plot_prompt.md` — 绘图提示词

**当前策略：** 保留现有工具，不扩展。未来具体项目的图件跟随项目文件夹（`projects/<项目名>/figures/`）。

### 6. 实验与结果库 — 暂缓

**当前内容：** `references/14_experiment_reproducibility.md`（实验可复现性规范，仅规则定义）

**为什么暂缓：** 无实际实验记录。实验和结果属于具体科研项目（`projects/<项目名>/experiments/`、`results/`）。入学后有实际实验时在项目文件夹中建。

### 7. 论文写作库 — 已有

**对应路径：** `write/`（30+文件：7脚本 + 7参考 + 13提示词 + 4检查清单 + 3模板 + 5子Skill + 配置）

**入口：** `write/README_速查.md`

**当前策略：** 保留观察，不扩展。入学前不使用复杂写作功能。入学后写课程论文/小论文时启用。

### 8. 组会 PPT 库 — 已有

**对应路径：** `组会/`（25+文件：2脚本 + 5规则 + 6模板 + 5检查清单 + 5桥接 + 3提示词 + ppt-speech-writer）

**入口：** `组会/README_速查.md`

**当前策略：** 保留观察，不扩展。入学前不使用。入学后每周组会时启用。组会 PPT 终稿保存到具体项目文件夹。

### 9. 导师沟通与任务管理 — 归 Personal-Brain

**academic-workflow 中相关内容：** `references/20_advisor_topic_fit.md`（导师课题适配规则）

**为什么归 Personal-Brain：** 导师真实沟通记录、任务跟踪、反馈整理涉及个人信息和决策记录，应放在总控层 Personal-Brain 中。academic-workflow 只保留公开的适配规则。在 Personal-Brain 中维护 `导师沟通与任务追踪.md`。

### 10. 项目复盘与作品集 — 归 Personal-Brain + 项目文件夹

**academic-workflow 的角色：** 为项目提供论文、方法、写作、组会、作图支持。不存项目复盘和作品集。

**复盘放哪里：** 每个项目结束后，在 Personal-Brain `projects/<项目名>.md` 中更新控制状态和复盘段落。
**作品集放哪里：** 在 Personal-Brain `career/作品集索引.md` 中维护跨项目作品集索引，指向各项目文件夹中的可展示成果。

## 不能乱动的目录

| 目录/文件 | 原因 |
|-----------|------|
| `scripts/` | 核心自动化脚本，修改可能破坏定时任务 |
| `paper_sources/` | 论文源文件归档，修改可能丢失阅读记录 |
| `reports/daily/` | 历史日报，修改破坏阅读记录连续性 |
| `logs/` | SQLite 数据库和运行日志 |
| `data/fulltext_papers/` | 已下载论文全文 |
| `references/` | 30个规则文件，是 15 阶段工作流的操作手册 |
| `backup_library/` | 备份快照，修改可能破坏回滚能力 |
| `SKILL.md` | 总控 Skill 定义，修改影响 Skills 路由 |
| `长期知识库.md` | 论文知识沉淀，修改可能丢失数据 |
| `待补读队列.md` | 待补读追踪 |

## 每日 / 每周 / 每月使用流程

### 每日（自动化 + 5 分钟扫一眼）

1. 12:00 — `daily_paper_curator.py` 自动搜索、筛选、生成日报 → 微信推送
2. 看一眼微信推送，判断是否有需要关注的论文
3. 如有感兴趣的论文，打开 `reports/daily/` 查看日报全文
4. 19:50 — Daily Supervisor 自动全项目监管 → 微信推送（Personal-Brain 侧）

### 每周（手动，30 分钟）

1. 打开 `长期知识库.md`，回顾本周新增论文
2. 从长期知识库中提取 3-5 条可复用方法 → 写入 `方法库索引.md`
3. 检查 `待补读队列.md`，判断是否需要手动获取某篇论文全文
4. （可选）提取 3-5 条写作句式 → 写入 `write/outputs/`

### 每月（手动，1 小时）

1. 回顾本月论文阅读趋势
2. 更新 Personal-Brain 中的项目控制状态
3. 更新 Personal-Brain `career/作品集索引.md`
4. （可选）写 3-5 条 lessons 到 Personal-Brain `lessons/`

## 10 个板块之间的信息流

```
论文检索 (arXiv + Semantic Scholar)
    │
    ▼
daily_paper_curator.py  ──→ 论文阅读库 (板块2)
    │                        ├── paper_sources/ (源文件+note复盘)
    │                        ├── reports/daily/ (日报)
    │                        └── 长期知识库.md (知识积累)
    │
    ├── 方法提取 ──→ 方法库索引.md (板块4)
    │
    ├── 写作需求 ──→ write/ (板块7)
    │
    ├── 组会准备 ──→ 组会/ (板块8)
    │
    └── 图表需求 ──→ tools/origin_auto_plot/ (板块5)

项目启动时：
    方法库索引.md ──→ 具体项目文件夹
    write/         ──→ 项目论文草稿
    组会/          ──→ 项目 PPT

项目结束后：
    项目文件夹 ──→ Personal-Brain 项目复盘
    项目文件夹 ──→ Personal-Brain 作品集索引
```

## 最小使用方式

**如果你只有 5 分钟：** 打开 `reports/daily/` 看今天日报。

**如果你有 15 分钟：** 看日报 + 打开 `长期知识库.md` 浏览最近一周新增。

**如果你想找方法：** 打开 `方法库索引.md`。

**如果你想写论文：** 打开 `write/README_速查.md` → 按最小流程操作。

**如果你要准备组会：** 打开 `组会/README_速查.md` → 按最小流程操作。

**如果你想理解整个系统：** 读本文件（你正在读）。

## 牢记

- **10 个板块是分类框架，不是 10 个目录。** 不要为了凑整新建文件夹。
- **academic-workflow 是执行层，Personal-Brain 是总控层。** 各司其职。
- **数据、实验、结果、图件跟项目走。** 不散落在 academic-workflow 根目录。
- **当前阶段：稳定验证，不扩建。** 入学前不新增子系统。
